# Optimized Prompt for Chuck Norris Quotes Project

## Project Overview
Create a Python-based project to scrape Chuck Norris quotes from various online databases and generate random quotes based on the scraped data. The project should be well-structured, testable, and follow best practices for Python development.

## Initialization Steps
- Initialize a Git repository in the project root.
- Create the following directory structure:
  - `.github/workflows/` (for CI/CD pipelines)
  - `.github/copilot-instructions.md` (for custom Copilot instructions)
  - `scripts/` (for utility scripts)
  - `download/` (for the scraping script)
  - `quotes/` (for the quote generation script)
  - `tests/` (for unit tests)
- Set up pre-commit hooks with a `.pre-commit-config.yaml` file.

## Tool Installation
- Install the latest versions of:
  - Pre-commit
  - Python (3.9 or higher)
  - Pytest
  - Coverage.py (for code coverage)

## Python Application Standards
For all Python scripts and modules:
- Use `pyproject.toml` for project configuration and dependencies.
- Specify latest stable versions as minimum required versions in `requirements.txt` and `pyproject.toml`.
- Write code that is fully unit testable.
- Create parameterized unit tests covering happy path and edge cases.
- Run tests automatically and ensure they pass.
- Achieve at least 95% code coverage.
- Implement parameter validation with comprehensive help options (-h, --help) that include usage instructions and examples.

## Script 1: Quote Scraper (`download/scraper.py`)
Create a Python script to scrape Chuck Norris quotes from multiple online databases.
- Perform ETL (Extract, Transform, Load) operations to handle various data formats (JSON, CSV, HTML, etc.).
- Output a single, efficient database file (e.g., SQLite) optimized for storage and quick access.
- Support the following command-line parameters:
  - `--sources` or `-s`: List of URLs or sources to scrape (default: predefined list of popular Chuck Norris quote sites)
  - `--output` or `-o`: Output database file path (default: `quotes.db`)
  - `--format` or `-f`: Output format (default: SQLite)
  - `--verbose` or `-v`: Enable verbose logging
  - `--help` or `-h`: Display help and usage examples
- Include error handling for network issues, invalid data, and rate limiting.
- Add logging for scraping progress and any issues encountered.

## Script 2: Quote Generator (`quotes/generator.py`)
Create a Python script to generate random Chuck Norris quotes using the scraped database.
- Generate up to 10,000,000 unique random quotes.
- Support the following command-line parameters:
  - `--count` or `-c`: Number of quotes to generate (default: 1, max: 10,000,000)
  - `--seed` or `-s`: Random seed for reproducible output (default: None for truly random)
  - `--output` or `-o`: Output file path (default: stdout)
  - `--format` or `-f`: Output format (options: text, json, csv; default: text)
  - `--database` or `-d`: Path to the quotes database (default: `download/quotes.db`)
  - `--verbose` or `-v`: Enable verbose logging
  - `--help` or `-h`: Display help and usage examples
- Ensure quotes are generated randomly but can be reproduced with a seed.
- Handle cases where the database is empty or insufficient quotes exist.
- Include performance optimizations for large quote counts.

- Use type hints and docstrings for all functions and classes.
- Ensure the code is cross-platform compatible (Windows, macOS, Linux).
- Add integration tests for the full ETL and generation pipeline.
- Consider using libraries like `requests` for scraping, `sqlite3` for database, `argparse` for CLI, and `logging` for output.
